# Account CRM — 종합 리포트 탭 개발 스펙
> Phase 3 기능 | Claude Code 작업용 지시서
> 전제: Phase 1 (고객목록·상세카드·Activity Log) 및 Phase 2 (Import 연동) 완료 상태

---

## 1. 전체 개념

보고서를 별도로 작성하지 않는다. CRM에 이미 쌓인 데이터를 그대로 집계·렌더링하여 주간/월간 보고서를 자동 생성한다.

### 데이터 소스 매핑

| 보고서 항목 | 데이터 소스 | 성격 |
|---|---|---|
| 수주·매출 실적 | 영업현황 Import 컬렉션 | 수시 업데이트 |
| 팀별 목표 / 달성률 | 사업계획 Import 컬렉션 | 연 1회 고정 |
| 팀별 이슈사항 | Activity Log (유형 태그 기반) | 담당자 실시간 입력 |
| Cross-selling 활동 | Activity Log — `cross_selling` 유형 | 담당자 실시간 입력 |
| 신규 계약 체결 | Activity Log — `contract` 유형 | 담당자 실시간 입력 |
| 고객 컨택 현황 | Activity Log — 전체 (날짜·담당자 기준) | 담당자 실시간 입력 |

---

## 2. Activity Log 유형 태그 (기존 정의 기준)

아래 유형이 이미 CRM에 정의되어 있음. 리포트는 이 태그를 필터 기준으로 사용한다.

```
general_contact    — 일반 컨택
price_negotiation  — 가격협의
quality_claim      — 품질 클레임
bidding            — 입찰
sample_request     — 샘플 요청
contract           — 계약 (신규 체결 / 갱신)
regulatory         — 규제·인증
market_info        — 시장정보 수집
cross_selling      — Cross-selling 활동
issue_sales        — 영업 이슈 (수주 관련)
issue_support      — 고객지원 (유관부서 협조 필요)
issue_quality      — 품질 이슈
other              — 기타
```

담당자(team) 필드: `overseas` / `domestic` / `sales_support`

## 3. 주간 리포트

### 3-1. 화면 구성

URL: `/report/weekly`

상단에 주차 선택기 (기본값: 현재 주)
→ `◀ 이전 주  |  4월 2주차 (4/7 ~ 4/13)  |  다음 주 ▶`

---

### 3-2. 섹션 구성

#### 섹션 A — 매출·수주 현황 (자동)

```
■ 1. 매출·수주 현황                              [단위: 백만원 / %]

┌─────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│         │ 전주 누적 │ 금주 신규 │ 당월 누적 │  당월 목표 │  달성률  │
├─────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ 해외(본사)│          │          │          │          │          │
│ BPU     │          │          │          │          │          │
│ 국내    │          │          │          │          │          │
│ 합계    │          │          │          │          │          │
└─────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
  ※ 매출: Import 실적 기준 / 목표: 사업계획 Import 고정값

┌─────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  수주   │ 전주 누적 │ 금주 신규 │ 당월 누적 │  당월 목표 │  달성률  │
├─────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ 해외(본사)│          │          │          │          │          │
│ BPU     │          │          │          │          │          │
│ 국내    │          │          │          │          │          │
│ 합계    │          │          │          │          │          │
└─────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

**데이터 로직:**
- "금주 신규" = Import 데이터에서 해당 주차(월~일) 범위 필터
- "당월 누적" = Import 데이터에서 당월 1일 ~ 리포트 기준일 합계
- "당월 목표" = 사업계획 Import에서 해당 월 팀별 목표값 (고정)
- "달성률" = 당월 누적 ÷ 당월 목표 × 100

---

#### 섹션 B — 이슈사항 (Activity Log 자동 집계)

```
■ 2. 영업본부 주간 이슈사항

┌──────────────┬──────┬──────────┬──────────────────────────────────────────┬──────┬────────┐
│ 구분         │ 팀   │ 고객명   │ 주요 내용                                 │ 담당 │ 상태   │
├──────────────┼──────┼──────────┼──────────────────────────────────────────┼──────┼────────┤
│ 영업이슈     │ 해외 │          │ [Activity Log issue_sales 내용 자동 인용] │      │ Open   │
│ (수주관련)   │ 국내 │          │                                          │      │        │
│              │ 지원 │          │                                          │      │        │
├──────────────┼──────┼──────────┼──────────────────────────────────────────┼──────┼────────┤
│ 고객지원     │ 해외 │          │ [issue_support 내용 자동 인용]            │      │        │
│ (유관부서    │ 국내 │          │                                          │      │        │
│  협조필요)   │ 지원 │          │                                          │      │        │
├──────────────┼──────┼──────────┼──────────────────────────────────────────┼──────┼────────┤
│ 품질이슈     │ 해외 │          │ [issue_quality 내용 자동 인용]            │      │        │
│              │ 국내 │          │                                          │      │        │
├──────────────┼──────┼──────────┼──────────────────────────────────────────┼──────┼────────┤
│ 기타         │ 해외 │          │ [other 내용 자동 인용]                    │      │        │
│              │ 국내 │          │                                          │      │        │
└──────────────┴──────┴──────────┴──────────────────────────────────────────┴──────┴────────┘
```

**데이터 로직:**
- 해당 주(월~일) 내 등록된 Activity Log를 유형 태그 기준으로 자동 분류
- 각 행 = Activity Log 1건 (고객명, 내용, 담당자, 상태 자동 인용)
- 내용이 없는 팀/구분 행은 빈 행으로 표시 (삭제하지 않음 — 보고 양식 통일성 유지)
- 상태 배지: Open(빨강) / In Progress(노랑) / Closed(초록)

### 3-3. Export 기능

- 화면 우상단 `Excel 다운로드` 버튼
- 파일명: `영업본부_주간회의_보고자료_4월2주차_v1_날짜.xlsx`
- 시트 구성: `주간종합_(4월2W)` 단일 시트 (현재 Excel 양식과 동일한 레이아웃)
- 기존 Sheet1(수주 Raw) 포함 여부: **포함하지 않음** (보고용만 Export)

## 5. UI 공통 사항

- 리포트 탭은 좌측 네비게이션에 `📊 리포트` 메뉴로 추가
- 주간/월간 전환: 탭 내 상단 토글 버튼
- 화면 렌더링과 Excel Export 레이아웃 최대한 동일하게 유지
- 숫자 단위: 화면은 "백만원" 표시, Export는 원화 원단위 (Import 원본 그대로)
- 달성률 색상 코딩: ≥100% 파랑 / 80~99% 검정 / <80% 빨강

## 6. Firebase 쿼리 설계 (참고)

```javascript
// 주간 Activity 집계 (이슈 섹션용)
db.collection('activities')
  .where('date', '>=', weekStart)
  .where('date', '<=', weekEnd)
  .where('type', 'in', ['issue_sales', 'issue_support', 'issue_quality', 'other'])
  .orderBy('team')
  .orderBy('date', 'desc')

// 당월 수주 실적 (매출현황 섹션용)
db.collection('sales_import')
  .where('order_date', '>=', monthStart)
  .where('order_date', '<=', monthEnd)
  .where('status', '==', '수주확정')

// 목표값 조회 (사업계획 — 고정)
db.collection('annual_plan')
  .where('year', '==', currentYear)
  .where('team', 'in', ['overseas', 'bpu', 'domestic'])
```

## 7. 개발 우선순위

1. **주간 리포트 — 섹션 A (매출·수주 수치)** 먼저 구현 → 수치 자동화가 핵심
2. **주간 리포트 — 섹션 B (이슈사항 자동 집계)** → Activity Log 태그 기반
3. **Excel Export** → 기존 Excel 양식 레이아웃 최대한 유지
4. **월간 리포트 — 섹션 B·C·D** → 주간 로직 재사용
5. **월간 리포트 — 섹션 A·E (반자동 입력 필드)** → 마지막
